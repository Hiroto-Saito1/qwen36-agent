# work README

`work/` は一時ファイル、動画素材、抽出フレーム、探索キャッシュを置く場所です。基本的にはGit管理しません。このREADMEだけ例外的に残します。

## 役割

- `outputs/` は残したい成果物。
- `work/` は成果物を作るための素材・中間生成物・キャッシュ。
- `models/` は `workspace/` の外にあり、GGUFモデル本体と軽量検索モデルのキャッシュを置く場所。

## 主なサブディレクトリ

### `work/free-video-test/`

Big Buck Bunny検証用の動画素材を置きます。

想定されるファイル:

- `big_buck_bunny_320x180.mp4`
  - Blender公式の元動画。
- `big_buck_bunny_5min.mp4`
  - 検証用に先頭5分へ切った動画。

これらは `outputs/free-video-test/.../reproduce.sh` が必要に応じて作ります。

### `work/video-event-search/`

`video_event_search.py` のキャッシュ置き場です。

中には設定ハッシュごとのディレクトリが作られます。各ディレクトリには、おおむね次のものが入ります。

- 抽出したフレーム
- 軽量検索用に切り出したスキャンフレーム
- モデルからの生レスポンス
- 正規化済み判定
- `config.cache-identity.json`

同じ入力JSON、同じ動画、同じモデル、同じ検索モデル、同じスクリプト版で再実行した場合、このキャッシュを使って続きから進みやすくなります。

条件や探索ロジックを変えて結果を取り直したい場合は、該当するキャッシュディレクトリを消して構いません。

### `work/vl-smoke-test/`

VLモデルの簡易動作確認に使う動画やフレームを置きます。

これは「モデルが画像を読めるか」を見るための一時領域です。成果物として残したい結果は `outputs/vl-smoke-test/` 側に置きます。

## 消してよいもの

基本的に `work/README.md` 以外は再生成可能です。

特に次は消して構いません。

- 抽出フレーム
- ダウンロード途中の一時ファイル
- 古い探索キャッシュ
- 検証用に切り出した動画

ただし、大きな動画を再ダウンロードしたくない場合は `work/free-video-test/` の元動画だけ残しておくと楽です。

## 残すべきもの

長く残したい結果は `work/` ではなく `outputs/` に置きます。

例:

- `output.json`
- `event-search.md`
- `search-trace.json`
- 再現用 `input.json`
- 再現用 `reproduce.sh`

## よくある注意

- `work/` の中に最終成果物を置くと、あとでどれが重要かわからなくなります。
- `outputs/` の中に `reproduce.sh` を置き、同じディレクトリに結果を作る運用にすると、成果物と再現手順が対応しやすくなります。
- 動画イベント探索は同じ出力ディレクトリで再実行できます。既知の生成物だけを消して作り直すため、古い `config.snapshot.json` が残っていても通常は止まりません。
